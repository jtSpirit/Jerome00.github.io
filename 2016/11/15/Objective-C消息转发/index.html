Objective-C消息转发

一、消息转发机制

当向某个对象发送一条消息时，如果该对象的方法列表以及它相应的继承链上的方法列表中都无法找到该消息的方法实现时，则会出发消息转发机制。

执行消息转发机制相当于在该对象无法找到消息实现时的一种补救方式，也可以理解为防止程序crash的备选方案，系统为此提供了三种方案：

方案一：

+(BOOL)resolveInstanceMethod:(SEL)sel

方案二：

-(id)forwardingTargetForSelector:(SEL)aSelector

方案三：

-(NSMethodSignature *)methodSignatureForSelector:(SEL)aSelector;

-(void)forwardInvocation:(NSInvocation *)anInvocation;

三种方法的顺序执行，举例：如果方案一实现了，那就不会执行方案二。

1、动态方法解析

+(BOOL)resolveInstanceMethod:(SEL)sel;

首先当消息接受的对象无法找到方法实现时，运行时系统会调用该函数给对象一次机会来添加相应方法的实现，如果用户在该函数中动态的添加了相应方法的实现，则会跳转到方法的实现。

通过代码演示，创建三个类分别为dog1，dog2，dog3，分别对应三种方案。 在main.m调用其jump方法。

方案一dog1动态添加方法的实现：

#import "dog1.h"
#import <objc/runtime.h>

void jump(id self,SEL cmd);

@implementation dog1

+(BOOL)resolveInstanceMethod:(SEL)sel{
    if ([NSStringFromSelector(sel) isEqualToString:@"jump"]) {
        class_addMethod(self, sel, (IMP)jump, "v@:");
        return YES;
    }
    return [super resolveInstanceMethod:sel];
}

void jump(id self,SEL cmd){
    NSLog(@"%@ jump",self);
}

@end

2.备援接受者

-(id)forwardingTargetForSelector:(SEL)aSelector;

如果运行时在消息转发第一步中未找到调用方法的实现，那么接受者会调用上边的方法，该方法返回一个能处理该消息的对象，运行时会对对象方法列表进行查找，如找到则跳转到相应的方法实现，消息转发结束。

#import "dog2.h"
#import "dog1.h"

@implementation dog

+(BOOL)resolveInstanceMethod:(SEL)sel{
    return NO;
}

-(id)forwardingTargetForSelector:(SEL)aSelector{

    dog1 *c=[dog1 new];
    return c;
}

@end

方案二中先对方案一中的方法返回NO，运行时就会调用第二个方法。通过控制台可以看到jump方法的调用。

3.完整的消息转发

-(void)forwardInvocation:(NSInvocation *)anInvocation;

当运行时系统检测到第二步中用户未返回对象时，那么就要启动完整的消息转发机制了。

#import "dog3.h"
#import "dog1.h"

@implementation chiken

+(BOOL)resolveInstanceMethod:(SEL)sel{
    return NO;
}

-(id)forwardingTargetForSelector:(SEL)aSelector{
    return nil;
}

-(NSMethodSignature *)methodSignatureForSelector:(SEL)aSelector{

    if ([NSStringFromSelector(aSelector) isEqualToString:@"jump"]) {
        return [NSMethodSignature signatureWithObjCTypes:"v@:"];

    }
    return [super methodSignatureForSelector:aSelector];
}

-(void)forwardInvocation:(NSInvocation *)anInvocation{

 //改变调用对象   
//    [anInvocation invokeWithTarget:[[dog1 alloc]init]];
//改变选择子
    anInvocation.selector=@selector(run);
    [anInvocation invokeWithTarget:self];
}
-(void)run{

    NSLog(@"run");
}

@end

方案三必须覆盖两个方法，消息转发有两种选择，一种是改变调用对象，一种是改变方法选择子。
